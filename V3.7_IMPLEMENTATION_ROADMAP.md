## Sprint 1: Glimfático Clearance (Adenosina)

### Problema
Adenosina acumula durante vigília, mas não é "resetada" explicitamente em sono. Biologicamente, o sistema glimfático ativa durante sono profundo (N3) e limpa metabólitos.

### Solução
Adicionar reset explícito de adenosina durante N2 (revisão/poda) e N3 (REM).

### Código — Passo 1: Estender NeuroChem

**Arquivo:** `src/neurochem.rs`

```rust
impl NeuroChem {
    /// Representa ATP cerebral simulado (proxy para energia disponível)
    pub atp_pool: f32,  // [0.0, 1.0] — 1.0 = totalmente carregado
    
    /// Histórico de consumo ATP (para retroalimentação)
    pub atp_deficit_ms: f32,  // ms de deficit acumulado
}

impl NeuroChem::new() {
    // ... existing fields ...
    atp_pool: 1.0,
    atp_deficit_ms: 0.0,
}

impl NeuroChem::update(&mut self, sensor: &mut HardwareSensor, config: &Config) {
    // ... existing neurotransmitter updates ...
    
    // ── ATP Dynamics (Novo) ───────────────────────────────────
    // Durante vigília: ATP degrada baseado em carga CPU
    let atp_consumption_rate = (sensor.get_context_switches_per_sec() / 10000.0).clamp(0.0, 1.0);
    self.atp_pool -= atp_consumption_rate * 0.001;  // ~0.1% por tick em vigília alta
    self.atp_pool = self.atp_pool.max(0.0);  // não pode ser negativo
    
    // ATP deficit acumula — pressão para sono
    if self.atp_pool < 0.5 {
        self.atp_deficit_ms += (0.5 - self.atp_pool) * 10.0;  // ms de deficit
    }
    
    // Adenosina correlata com ATP deficit
    let adenosine_from_atp = (self.atp_deficit_ms / 1000.0).clamp(0.0, 1.0);
    let target_adenosine_new = (adenosine_from_atp * 0.7 + adenosine_load * 0.3).clamp(0.0, 1.0);
    self.adenosine = self.adenosine * 0.95 + target_adenosine_new * 0.05;
}

/// Glimfático clearance — chamado em sono N2/N3
pub fn glimfatico_clearance(&mut self) {
    // ATP regenera durante sono
    self.atp_pool = (self.atp_pool + 0.2).min(1.0);  // recupera 20% por ciclo de sono
    
    // Adenosina clearance proporcional a ATP regenerado
    self.adenosine *= 0.4;  // 60% redução por ciclo de sono
    
    // Deficit zerado
    self.atp_deficit_ms = 0.0;
}
```

## Sprint 2: REM Reverso (Reconsolidação Bidirecional)

### Problema
Reconsolidação existe, mas REM simplesmente "recombina" conexões forward. Biologicamente, REM envolve **replay reverso** (pós→pré) para permitir "unlearning" e reorganização.

### Solução
Adicionar fase de replay reverso em N3 que desativa sinapses lábeis seletivamente.

### Código — Passo 1: Estender MemoriaLabil

**Arquivo:** `src/storage/reconsolidacao.rs`

```rust
#[derive(Debug, Clone)]
pub struct MemoriaLabil {
    // ... existing fields ...
    
    /// Contador de vezes que foi reativada em REM reverso
    pub rem_reversals: u32,
    
    /// Último tempo de reativação (para controlar reversal rate)
    pub ultimo_reversal_ms: f32,
}

impl MemoriaLabil {
    /// Aplica reversal durante REM — permite "unlearning" ou associação nova
    pub fn reversal_rem(&mut self, dt_s: f64, rem_drive: f32) -> bool {
        if self.estado != EstadoMemoria::Reconsolidando {
            return false;
        }
        
        // REM reversal ocorre a ~10 Hz (100ms entre re-ativações)
        // Se last_reversal < 100ms, skip
        if self.ultimo_reversal_ms < 100.0 {
            return false;
        }
        
        // Aplicar pequena dedução (permite nova aprendizagem)
        // Baseado em Stickgold: REM permite "forgetting" de detalhes
        let reversal_factor = (1.0 - rem_drive * 0.05).clamp(0.0, 1.0);
        self.peso_atual *= reversal_factor;
        
        self.rem_reversals += 1;
        self.ultimo_reversal_ms = 0.0;
        self.modificacoes.push(ModificacaoLabil::Atualizacao {
            novo_contexto: format!("REM_reversal_{}", self.rem_reversals),
        });
        
        true
    }
}
```

### Código — Passo 2: Estender FaseSono::N3

**Arquivo:** `src/sleep_cycle.rs`

```rust
async fn fase_n3(&mut self, memoria: &mut MemoryTier) {
    println!("\n🌙 FASE 3: {}", FaseSono::N3.nome());
    
    // ── REM Reverso (Novo) ─────────────────────────────────
    // Durante ~30 min de "REM", reativar memórias lábeis em ordem reversa
    // Permitir "experimentação" — unlearning de associações instáveis
    
    if let Some(grafo_arc) = &self.grafo {
        let mut grafo = grafo_arc.lock().await;
        
        // REM drive: quanto mais lábil, mais reversal
        let labil_count = grafo.grafo_completo.conexoes.values()
            .filter(|c| c.marcador_poda < 0.5)  // proxy para "lábil"
            .count() as f32;
        let rem_drive = (labil_count / 100.0).clamp(0.0, 1.0);
        
        println!("   😴 REM Reverso: {} memórias lábeis, drive={:.2}", 
                 labil_count as usize, rem_drive);
        
        // Aplicar reversals a ~5 memórias aleatórias por ciclo
        let n_reversals = ((labil_count * 0.1) as usize).max(1);
        for _ in 0..n_reversals {
            // Selecionar memória lábil aleatória
            let ids: Vec<_> = grafo.grafo_completo.conexoes.keys().collect();
            if !ids.is_empty() {
                use rand::seq::SliceRandom;
                if let Some(&&id) = ids.choose(&mut thread_rng()) {
                    if let Some(c) = grafo.grafo_completo.conexoes.get_mut(&id) {
                        // Aplicar pequena dedução
                        c.peso = (c.peso * (1.0 - rem_drive * 0.03)).max(0.01);
                        c.marcador_poda += rem_drive * 0.02;  // marcar como "revisado"
                    }
                }
            }
        }
    }
    
    println!("   ✅ REM Reverso completado");
}
```

## Sprint 3: Social Reward (Oxitocina)

### Problema
Oxitocina modula apenas fear (BLA). Não há pathway para **social valência** — feedback de rejeição vs aceitação.

### Solução
Adicionar social_valence ao neurochem; ligar a oxitocina de forma bidirecional (recompensa social ↑ oxitocina ↑ BLA inibição).

### Código — Passo 1: Estender NeuroChem

**Arquivo:** `src/neurochem.rs`

```rust
pub struct NeuroChem {
    // ... existing fields ...
    
    /// Valência social [-1.0, 1.0]
    /// Negativa = rejeição/isolamento; Positiva = aceitação/pertencimento
    pub social_valence: f32,
}

impl NeuroChem::new() {
    // ... existing ...
    social_valence: 0.0,  // neutro por padrão
}

impl NeuroChem::update(&mut self, sensor: &mut HardwareSensor, config: &Config) {
    // ... existing updates ...
    
    // ── Social Valence (Novo) ──────────────────────────────
    // Em Selene, social_valence é fornecido externamente (via chat feedback)
    // Decay natural para neutro
    self.social_valence *= 0.98;  // meia-vida ~50 ticks
    
    // Oxitocina correlata com social valence
    let oxytocin_from_social = (self.social_valence.max(0.0) * 0.5).clamp(0.0, 1.0);
    self.oxytocin = self.oxytocin * 0.95 + oxytocin_from_social * 0.05;
    
    // Cortisol reduzido quando oxitocina alta (social support buffering)
    let cortisol_reduction = self.oxytocin * 0.3;
    let target_cortisol_new = (target_cortisol - cortisol_reduction).max(0.0);
    self.cortisol = self.cortisol * 0.98 + target_cortisol_new * 0.02;
}

/// Interface para feedback social (chamado via WebSocket)
pub fn registrar_feedback_social(&mut self, feedback: f32) {
    // feedback: -1.0 (rejeição) a +1.0 (aceitação)
    self.social_valence = feedback.clamp(-1.0, 1.0);
}
```
